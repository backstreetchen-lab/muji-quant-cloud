# MujiQuant Cloud - 创新药企量化分析技能

基于五智能体（5-Agent）分析框架，对港股/A股创新药企进行多维度量化分析，结合 EPS/营收预测，输出专业投资建议报告。

---

## 📐 分析框架

### 五智能体体系（0-10 分制）

| Agent | 职责 | 核心评估维度 |
|:---:|:---:|:---|
| 🧬 基本面 Agent | 财务健康与盈利质量 | 营收增速、盈利状态、EPS质量、毛利率、ROE、营收预测可信度 |
| 📈 技术面 Agent | 价格趋势与量价关系 | 趋势方向、均线系统、量价配合、目标价空间、技术风险信号 |
| 💰 资金面 Agent | 资金流向与机构态度 | 南向资金、机构持仓、回购信号、融资能力、成交活跃度 |
| 🌍 宏观情绪 Agent | 行业政策与市场情绪 | ASCO/ESMO催化、医保政策、行业Beta、利率环境、地缘风险 |
| 🛡️ 风控 Agent | 风险识别与仓位管理 | 估值风险、管线风险、流动性风险、地缘风险、集中度风险 |

### 附加分析维度

| 维度 | 说明 |
|:---:|:---|
| 📊 EPS 预测 | 当年实际 EPS vs 下一年一致预期 EPS，增速排名与可信度评估 |
| 💰 营收预测 | 当年营收 vs 下一年营收预测（券商一致预期/公司指引），核心增长驱动识别 |

### 评分加权

```
综合评分 = 基本面×0.30 + 技术面×0.20 + 资金面×0.15 + 宏观情绪×0.15 + 风控×0.20
```

### 评级映射

| 综合评分 | 评级 | 操作建议 |
|:---:|:---:|:---|
| ≥ 7.5 | ⭐ 强烈推荐 | 核心仓位配置 |
| 6.5 - 7.4 | ✅ 推荐 | 标准仓位配置 |
| 5.5 - 6.4 | 👀 关注 | 小仓位/等待催化 |
| 4.5 - 5.4 | ⚠️ 观望 | 暂不介入 |
| < 4.5 | 🚫 规避 | 回避 |

## 🔄 标准分析流程

### Step 1: 数据采集（多源冗余）

按优先级依次尝试，确保数据可靠性：

1. **WebSearch** → 搜索公司最新财报、EPS、营收数据
2. **WebFetch** → 抓取东方财富/雪球/新浪财经等页面数据
3. **yfinance** → Python 库获取行情、市值、PE 等量化数据
4. **券商研报** → 搜索目标价、EPS预测、营收预测等一致预期

**数据校验规则**：
- 优先采用公司公告/年报的官方口径数据
- EPS 预测优先取多家券商一致预期均值
- 营收预测优先取公司官方指引，次取券商一致预期
- 行情数据以最新收盘价为准，需标注日期

### Step 2: 五智能体评分

每个 Agent 独立评分（0-10），基于数据给出判断依据：

- **基本面 Agent**：重点评估盈利拐点确认、营收梯队分化、EPS质量、营收预测可信度
- **技术面 Agent**：重点评估趋势方向、量价关系、目标价上行空间
- **资金面 Agent**：重点评估南向资金动向、机构覆盖度、回购信号
- **宏观情绪 Agent**：重点评估ASCO/ESMO催化、医保政策、行业Beta
- **风控 Agent**：重点识别估值风险、管线风险、流动性风险、地缘风险、集中度风险

### Step 3: EPS/营收预测深度分析

- 对比当年实际 vs 下一年预测，计算增速
- 评估预测可信度（机构覆盖数量、公司指引明确性）
- 识别核心增长驱动因素
- 标注不确定性来源

### Step 4: 生成投资建议

- 每只标的给出：定位、投资逻辑、操作建议（含仓位比例和入场价位）
- 按风险偏好给出组合配置方案（稳健型/平衡型/进取型）
- 列出关键催化时间线

### Step 5: 输出报告 ⬅️ **关键步骤**

## 📦 报告输出规则（必须执行）

### 规则 1：三通道输出

**每次分析完成后，必须同时输出以下三种格式：**

| 输出方式 | 格式 | 说明 | 工具 |
|:---:|:---:|:---|:---|
| 📄 HTML 文件 | `.html` | 交互式网页报告，支持动画/渐变等视觉效果 | `Write` 工具 |
| 📑 PDF 文件 | `.pdf` | 便于存档、转发、打印的静态文档 | `weasyprint` |
| 🔗 预览链接 | URL | 可直接点击查看的在线链接 | `preview` 技能 |

### 规则 2：文件命名规范

```
/workspace/mujiquant_{市场}_{主题}_{日期}.html
/workspace/mujiquant_{市场}_{主题}_{日期}.pdf
```

示例：
- `/workspace/mujiquant_hk_biotech_report.html`
- `/workspace/mujiquant_hk_biotech_report.pdf`
- `/workspace/mujiquant_hk_oncology_20260430.html`
- `/workspace/mujiquant_hk_oncology_20260430.pdf`

### 规则 3：PDF 生成流程

HTML 文件写入后，使用 `weasyprint` 将 HTML 转为 PDF：

```python
from weasyprint import HTML

html_path = '/workspace/mujiquant_xxx.html'
pdf_path = '/workspace/mujiquant_xxx.pdf'
HTML(filename=html_path).write_pdf(pdf_path)
```

**注意事项**：
- weasyprint 已预装在沙箱环境中，无需额外安装
- PDF 中 CSS 动画/渐变会自动转为静态渲染
- 如 weasyprint 渲染异常，降级使用 `xhtml2pdf`（同样预装）
- PDF 生成后检查文件大小，正常应 > 100KB

### 规则 4：部署预览链接

文件写入 `/workspace/` 后，**必须**调用 `preview` 技能部署预览链接：

```bash
# 1. 启动 HTTP 服务
lsof -ti:8000 | xargs kill -9 2>/dev/null || true
nohup python3 -m http.server 8000 --bind 0.0.0.0 --directory /workspace > server.log 2>&1 &
sleep 3

# 2. 调用 notify 生成预览 URL
<skill-directory>/../preview/notify 8000
```

### 规则 5：向用户回复格式

完成三通道输出后，按以下格式回复用户：

```
📄 HTML报告：/workspace/mujiquant_xxx.html
📑 PDF报告：/workspace/mujiquant_xxx.pdf

🔗 在线预览：[点击查看完整报告](<URL>)

如链接无法打开，复制以下网址到浏览器：
<URL>
```

### 规则 6：用户在小程序端

**用户可能通过微信小程序等移动端访问，无法直接打开本地文件路径。** 因此：
- **必须**提供预览链接，不能仅告知文件路径
- 预览链接是用户查看报告的主要方式
- PDF 文件供用户下载转发（微信可直接发送 .pdf 文件）
- HTML 文件作为备份，供桌面端交互式查看

## 📋 报告模板结构

生成的 HTML 报告应包含以下板块（按顺序）：

1. **报告头部** — 标题、日期、分析框架标识
2. **投资价值综合排名** — 所有标的评分排名与评级
3. **五智能体评分总览** — 每只标的 5 个维度的评分条形图
4. **基本面核心数据一览** — 营收、净利、增速、毛利率对比表
5. **EPS 深度分析** — 当年实际 EPS 柱状图 + 下一年预测对比表
6. **营收预测与增长路径** — 每只标的的营收指引、增速、券商目标价
7. **五智能体深度分析** — 每个 Agent 的核心洞察（每个 Agent 3-5 条）
8. **个性化投资建议** — 每只标的的定位/逻辑/操作建议
9. **组合配置建议** — 稳健型/平衡型/进取型三档配置
10. **关键催化时间线** — ASCO、FDA审批、中报等关键节点
11. **免责声明** — 风险提示与免责

### 报告视觉规范

- 暗色主题（深色背景 `#0a0e17`）
- 五智能体配色：蓝/青/绿/紫/红
- 使用 CSS 渐变和动画提升视觉效果
- 响应式布局，适配移动端查看
- Noto Sans SC 字体

## ⚠️ 注意事项

1. **数据时效性**：所有数据需标注日期，优先使用最新财报/公告口径
2. **EPS 口径**：港股 EPS 单位为"元/股"或"分/股"，需统一换算
3. **亏损标的**：无 PE 时用 PS 或 DCF 估值，EPS 为负数时重点看亏损收窄趋势
4. **流动性风险**：小市值港股（日均成交 < 5000 万港元）需特别标注风险
5. **免责声明**：每份报告必须包含免责声明，明确不构成投资建议
