## 一、数据采集规则

### 1.1 数据源优先级

| 优先级 | 数据源 | 适用场景 | 可靠性 |
|:---:|:---|:---|:---|
| 1 | 公司公告/年报 | 营收、净利、EPS、毛利率等官方口径数据 | ⭐⭐⭐⭐⭐ |
| 2 | 券商研报/一致预期 | EPS预测、营收预测、目标价 | ⭐⭐⭐⭐ |

### 1.2 多源冗余策略

当主数据源不可用时，按优先级降级：

```
公司公告 → 券商研报 → 财经网站 → yfinance → WebSearch
```

### 1.3 数据校验

- 营收数据需与公司公告口径一致（注意"营业总收入"vs"产品收入"的区别）
- EPS 需区分"基本EPS"和"摊薄EPS"，统一使用基本EPS
- 港股 EPS 单位需统一换算为"元/股"（部分来源用"分/股"）
- 行情数据需标注日期（港股收盘价以当日为准）

## 三、EPS/营收预测分析规则

### 3.1 EPS 分析

1. 收集当年实际 EPS 和下一年预测 EPS
2. 计算增速：EPS增速 = (预测EPS - 实际EPS) / |实际EPS| × 100%
3. 当实际 EPS 为负时，标注"亏损收窄/扩大"而非计算增速百分比
4. 评估预测可信度：
   - 高可信：机构覆盖>15家，公司有明确指引
   - 中可信：机构覆盖5-15家
   - 低可信：机构覆盖<5家

### 3.2 营收预测分析

1. 优先采用公司官方指引（如药明生物2026年+13-17%指引）
2. 次采用券商一致预期均值
3. 识别核心增长驱动：新产品放量 vs BD授权 vs 价格提升 vs 市场扩张
4. 标注不确定性来源

## 四、投资建议规则

### 4.1 评级标准

| 综合评分 | 评级 | 操作建议 | 典型特征 |
|:---:|:---:|:---|:---|
| ≥ 7.5 | 强烈推荐 | 核心仓位(20-40%) | 持续盈利+估值合理+增长确定 |
| 6.5-7.4 | 推荐 | 标准仓位(10-20%) | 首次盈利+空间大或龙头溢价 |
| 5.5-6.4 | 关注 | 小仓位(3-10%) | 有催化但不确定性高 |
| 4.5-5.4 | 观望 | 暂不介入 | 亏损/估值偏高/流动性差 |
| < 4.5 | 规避 | 回避 | 亏损扩大+无催化+流动性极差 |

### 4.2 组合配置

- 稳健型：60%+ 配置于持续盈利标的（恒瑞/药明类）
- 平衡型：均衡配置，兼顾确定性和弹性
- 进取型：侧重首次盈利/催化驱动的弹性标的

### 4.3 操作建议模板

每只标的需包含：
- **定位**：一句话说明该标的在组合中的角色
- **逻辑**：2-3句核心投资逻辑（含EPS/营收预测数据支撑）
- **操作**：仓位比例建议 + 入场价位参考 + 关键跟踪指标

## 五、报告输出规则（强制执行）

### 5.1 三通道输出

**每次分析完成后，必须同时完成以下三步：**

1. **HTML 文件导出**：将交互式报告写入 `/workspace/` 目录
2. **PDF 文件导出**：使用 weasyprint 将 HTML 转为 PDF
3. **预览链接**：调用 `preview` 技能部署 HTTP 服务，生成可点击 URL

### 5.2 为什么必须三通道

- 用户可能通过微信小程序访问，**无法直接打开本地文件路径**
- 预览链接是移动端用户查看报告的主要方式
- **PDF 文件**便于存档、转发（微信可直接发送 .pdf）、打印
- HTML 文件作为备份，供桌面端交互式查看

### 5.3 PDF 生成流程

```python
from weasyprint import HTML

html_path = '/workspace/mujiquant_xxx.html'
pdf_path = '/workspace/mujiquant_xxx.pdf'
HTML(filename=html_path).write_pdf(pdf_path)
```

- weasyprint 已预装在沙箱环境，无需额外安装
- 如 weasyprint 渲染异常，降级使用 `xhtml2pdf`
- PDF 中 CSS 动画/渐变会自动转为静态渲染
- 生成后检查文件大小，正常应 > 100KB

### 5.4 部署预览链接

```bash
# 文件已写入 /workspace/ 后：

# 1. 启动 HTTP 服务
lsof -ti:8000 | xargs kill -9 2>/dev/null || true
nohup python3 -m http.server 8000 --bind 0.0.0.0 --directory /workspace > server.log 2>&1 &
sleep 3

# 2. 调用 preview 技能的 notify 生成预览 URL
/root/.codebuddy/skills/preview/notify 8000
```

### 5.5 回复格式

